# Project Roadmap

## 1. Purpose

This document defines the next coherent roadmap for UNV-HMND and Humanity Pro University.

The roadmap exists to prevent unnecessary complexity and keep the project aligned with its first principles.

## 2. Current Foundation

The project now has:

- WHY.md
- FIRST_PRINCIPLES.md
- EVOLUTION_PATH.md
- SYSTEM_RELATIONSHIP_MAP.md
- PROJECT_LAYERS.md
- RAIDER_HOLBERTON_OS_OPERATING_MODEL.md
- HOLBERTON_SPINE.md
- HUMANITY_PRO_UNIVERSITY_CORE.md
- HUMANITY_PRO_UNIVERSITY_RELATIONSHIPS.md

## 3. Current Phase

Current phase:

Architectural Foundation and Coherence.

Focus:

- documentation;
- coherence;
- relationships;
- operating model;
- foundations;
- human-centered vision.

## 4. Next Phase

Next phase:

Understanding Before Building.

Objective:

Review existing documents, reduce duplication, clarify relationships, and prepare the project before adding new technical capabilities.

## 5. Short-Term Roadmap

1. Review documentation coherence.
2. Identify duplicated concepts.
3. Connect existing architecture documents.
4. Create a documentation index.
5. Only after that, decide whether to move toward UI, agents, memory, or backend.

## 6. Medium-Term Roadmap

- Improve user-facing clarity.
- Connect the documentation to the app interface.
- Create visible project cards for the core concepts.
- Keep all capabilities proposal-only.
- Maintain human approval.

## 7. Long-Term Roadmap

- Controlled agents.
- Safe memory.
- Interview system.
- Learning paths.
- Community layer.
- Builder layer.
- Production only when governance, permissions, audit, and reversibility are mature.

## 8. Boundaries

Do not add complexity before understanding.

Do not create agents before governance is ready.

Do not connect private data without consent.

Do not claim AGI.

Do not claim artificial consciousness.

Do not execute without approval.

Proposal != Execution.

Human approval remains mandatory.
