# Screenshot Plan

This plan defines the official screenshot set for the public Humanity Guide OS showcase.

## Recommended Setup

- Desktop resolution: 1440 x 1200.
- Mobile sanity viewport: 390 x 844.
- Browser zoom: 100%.
- Theme: dark mode only.
- State: stable page state, no loading spinners, no broken placeholders.
- Console: clear before each capture and confirm no critical errors.

## Official Screenshot Order

1. `01-home-governance-contract.png`
   - Route: `/`
   - Show: homepage hero, recruiter story, governance contract, and primary CTAs.
   - Purpose: explain what Humanity Guide OS is in one screenshot.

2. `02-demo-safe-response.png`
   - Route: `/demo`
   - Show: public demo after one safe prompt response.
   - Purpose: demonstrate a real public interaction without external execution.

3. `03-lab-governance-simulation.png`
   - Route: `/lab`
   - Show: organization simulation, GENIO analysis, safety indicators, and proposal-only messaging.
   - Purpose: show governance-first product depth.

4. `04-lab-preview-guardrails.png`
   - Route: `/lab`
   - Show: controlled read-only preview or email preview guardrails.
   - Purpose: show preview-only boundaries and no real destructive actions.

5. `05-personal-owner-dashboard.png`
   - Route: `/personal`
   - Show: owner daily surface, dashboard status, organization zones, and manual checklist.
   - Purpose: show the owner-controlled daily experience.

## Naming Convention

Use:

```text
NN-route-subject.png
```

Examples:

```text
01-home-governance-contract.png
02-demo-safe-response.png
03-lab-governance-simulation.png
04-lab-preview-guardrails.png
05-personal-owner-dashboard.png
```

## What To Avoid

- do not show secrets, owner codes, tokens, or local environment values
- do not show browser extension noise
- do not show loading states
- do not show console errors
- do not crop out safety chips when they are part of the message
- do not imply real runtime, filesystem execution, Gmail integration, or autonomous execution

## Recruiter-Safe Visual Rules

- keep the navigation visible when it helps orientation
- keep safety language visible where relevant
- prefer calm, readable framing over dramatic close-ups
- show current capability, preview, or blueprint status clearly
- keep visual hierarchy premium, stable, and enterprise-ready
